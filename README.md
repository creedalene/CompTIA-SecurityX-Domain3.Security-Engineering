### Overview

These are my personal study notes and reference materials for **CompTIA SecurityX (CAS-005) Domain 3.0: Security Engineering**. The content emphasizes practical, post-graduate level application with heavy focus on real-world implementation in both enterprise and Department of Defense (DoD) environments.

### Repository Purpose

I built this repository to help senior cybersecurity professionals, security architects, and defense contractors prepare for the CAS-005 exam while serving as a living reference for day-to-day security engineering work. The material bridges theory and practice, showing how to troubleshoot complex issues, implement robust controls, and apply advanced techniques in production environments.

### Domain 3.0 Coverage

This domain makes up approximately **25%** of the SecurityX exam and focuses on the design, implementation, and troubleshooting of security solutions across identity, endpoints, networks, hardware, specialized systems, automation, and cryptography.

### Key Topics

* Advanced IAM troubleshooting (biometrics, conditional access, federation, PAM)
* Endpoint protection (EDR, application control, SELinux, attack surface reduction)
* Network security (DNSSEC, DMARC, TLS, IPS/IDS, ACLs)
* Hardware roots of trust (TPM, HSM, Secure Boot, Measured Boot)
* OT/ICS, IoT, embedded systems, and legacy system security
* Security automation (IaC, SOAR, SCAP, scripting)
* Post-quantum cryptography, homomorphic encryption, AEAD, and more
* Cryptographic use cases and implementation strategies

### DoD / Defense Focus

All notes include relevant DoD applications, including:
* Integration with RMF, DISA STIGs, and eMASS
* DoD PKI, CAC, and Zero Trust Reference Architecture
* IAVM, CJCSM 6510.01B, and CORA/CCRI readiness
* Protection of CUI and National Security Systems (NSS)

> **DoD Naming Note:** Content uses "Department of Defense (DoD)" for certification alignment and consistency with current exam materials and official documentation.
